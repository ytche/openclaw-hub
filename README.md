# openclaw-hub

跨平台 Agent 配置与协作管理中心。

通过 Git + GitHub 在多台机器间同步 Agent 配置、教程、规范和项目文档。

## 支持平台

| 平台 | 状态 | 说明 |
|------|------|------|
| OpenClaw | ✅ 已实现 | 首平台适配 |
| DeerFlow | 📝 规划中 | 占位待实现 |
| Claude Code | 📝 规划中 | 占位待实现 |

## 快速开始

```bash
# 克隆仓库
git clone https://github.com/<your-org>/openclaw-hub.git
cd openclaw-hub

# 验证配置格式
make validate

# 同步所有平台配置
make sync

# 同步特定平台
make sync-openclaw

# 同步特定机器实例
make sync-instance laptop

# 新机器初始化
make init INSTANCE_NAME=<机器名>
```

## 项目结构

```
openclaw-hub/
├── docs/              # 公共教程/文档
├── standards/         # 统一格式标准层
│   ├── schemas/       # 格式定义
│   ├── templates/     # 模板
│   ├── common/        # 通用规范
│   └── agents/        # Agent 级别配置
├── adapters/          # 平台适配转化层
├── shared/            # 公共工作区（项目协作）
├── instances/         # 各机器实例私有配置
└── scripts/           # 工具脚本
```

## 核心约定

### 配置优先级

```
instances > standards
```

当 `instances/<机器>/<平台>/agents/<agent>/` 中存在同名文件时，优先使用实例私有配置。

### 自动生成标记

所有由 adapter 自动生成的文件，头部包含注释：

```markdown
<!-- AUTO-GENERATED by openclaw-hub adapter -->
<!-- Source: standards/agents/<name>/xxx.md -->
<!-- DO NOT EDIT DIRECTLY; edit the source file instead -->
```

手工覆盖的文件（instances 中）如果与 source 不同，允许存在，但无此标记。

## 开发

### 添加新 Agent

1. 在 `standards/agents/` 下创建目录并编写统一配置
2. 运行 `make sync` 生成各平台适配文件
3. 如需特定机器覆盖，在 `instances/<机器>/<平台>/agents/<agent>/` 下创建文件

### 添加新平台适配

1. 在 `adapters/` 下创建平台目录
2. 实现 `adapter.py` 脚本
3. 在 `Makefile` 添加同步入口
