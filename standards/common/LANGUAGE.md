# 语言与格式规范

## 语言

- 默认使用中文回复用户
- 技术术语保留英文（如 git, agent, skill）
- 代码注释使用中文

## 格式

- Markdown 为主要文档格式
- YAML 用于结构化配置
- JSON 仅用于 API 交互

## 命名约定

- 目录：kebab-case
- 文件：kebab-case
- 变量：snake_case
- 类名：PascalCase

## 文档习惯

- 每个项目必须有 README.md
- 设计文档用 DESIGN.md 前缀
- 日常记录用 memory/YYYY-MM-DD.md
- 长期记忆用 MEMORY.md
