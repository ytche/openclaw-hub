# 文档编写习惯

## 文件组织

- **README.md**: 项目入口，必须包含：项目定位、快速开始、项目结构
- **DESIGN.md**: 设计文档，包含架构、约定、实施计划
- **TODO.md**: 任务清单
- **CHANGELOG.md**: 变更记录

## 写作风格

- 开门见山，结论先行
- 用列表和表格代替大段文字
- 代码示例必须可运行
- 链接使用相对路径

## 更新频率

- 设计文档随架构变更更新
- 任务清单每日/每周更新
- 变更记录每次 commit 更新

## 交叉引用

- 引用其他文档时使用相对路径
- 使用统一的术语表（见 LANGUAGE.md）
