# Analysis Agent 工作流规范

> 本规范适用于数据分析、投资分析、市场研究等分析类 Agent。
>
> 继承自 `standards/common/WORKFLOW.md` 的通用 5 阶段骨架，此处补充分析专属规则。

---

## 一、通用骨架回顾

所有 Agent 遵循：

```
入口 → 设计 → 执行 → 验证 → 收尾
```

本文件只覆盖 **执行** 和 **验证** 阶段的分析专属规则。

---

## 二、执行阶段

### 2.1 数据采集

| 原则 | 说明 |
|---|---|
| 多源交叉 | 同一指标从 ≥2 个来源采集，对比验证 |
| 来源标注 | 每条数据标注来源、采集时间 |
| 时效检查 | 检查数据更新频率，标注过期风险 |

### 2.2 数据清洗

```
原始数据 → 去重 → 补全缺失 → 格式统一 → 异常值标记 → 清洗后数据
```

- 缺失值处理策略必须说明（删除/填充/标记）
- 异常值单独标记，不直接删除（避免信息损失）
- 清洗规则写入日志，确保可复现

### 2.3 分析建模

| 步骤 | 说明 |
|---|---|
| 明确假设 | 列出分析的前提假设 |
| 选择方法 | 匹配问题类型（时序/分类/回归/聚类） |
| 参数记录 | 记录模型参数和选择理由 |
| 结果产出 | 生成报告/可视化/结论 |

### 2.4 敏感性检查

- 改变关键参数，观察结论是否稳健
- 检查边界条件（极端情况）
- 标注置信区间 / 误差范围

---

## 三、验证阶段

### 3.1 数据质量检查

- [ ] 样本是否有代表性（样本偏差检查）
- [ ] 数据是否完整覆盖目标范围
- [ ] 来源是否可靠、权威
- [ ] 时效性是否符合需求

### 3.2 逻辑检查

- [ ] 分析方法是否匹配问题类型
- [ ] 因果推断是否有混淆变量
- [ ] 相关性 ≠ 因果性，是否明确区分
- [ ] 结论是否超出数据支撑范围

### 3.3 偏见检查

- [ ] 数据来源是否有选择性偏见
- [ ] 分析框架是否有先入为主的假设
- [ ] 是否考虑了反面证据
- [ ] 结论措辞是否过度肯定（确定性 vs 概率性）

---

## 四、收尾阶段

### 4.1 交付物

| 类型 | 内容 |
|---|---|
| 执行摘要 | 核心结论，3-5 句话 |
| 详细分析 | 方法、数据、过程 |
| 数据来源 | 完整来源列表，含 URL/时间 |
| 风险提示 | 局限性、不确定性、建议的后续行动 |
| 附录 | 原始数据、代码、中间结果 |

### 4.2 记录

- 分析过程记录到 memory，便于追溯
- 关键假设变更记录原因
- 失败的分析路径也记录（避免重复踩坑）

---

## 五、快速参考

| 场景 | 流程 |
|---|---|
| 投资分析 | 数据采集 → 清洗 → 估值模型 → 敏感性分析 → 风险评估 → 报告 |
| 市场研究 | 多源采集 → 交叉验证 → 趋势分析 → 竞争格局 → 结论 |
| 数据报告 | 需求确认 → 数据采集 → 清洗 → 可视化 → 解读 → 交付 |
