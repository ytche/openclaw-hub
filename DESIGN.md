# openclaw-hub 设计文档

## 1. 项目定位

跨平台 Agent 配置与协作管理中心。通过 Git + GitHub 在多台机器间同步 Agent 配置、教程、规范和项目文档。

**支持平台（逐步扩展）：**
- OpenClaw
- DeerFlow（规划中）
- Claude Code（规划中）

## 3. 核心约定

### 3.1 配置合并优先级

```
instances > standards
```

即：当 `instances/<机器>/<平台>/agents/<agent>/` 中存在同名文件时，优先使用实例私有配置。

### 3.2 文件来源标记

所有由 adapter 自动生成的文件，头部必须包含注释标记：

```markdown
<!-- AUTO-GENERATED by openclaw-hub adapter -->
<!-- Source: standards/agents/liuyun/soul.md -->
<!-- DO NOT EDIT DIRECTLY; edit the source file instead -->
```

手工覆盖的文件（instances 中）如果与 source 不同，允许存在，但无此标记。

### 3.3 统一格式规范

**结构化配置 → YAML**
```yaml
# standards/agents/liuyun/agent.yaml
name: 闲云
creature: 璃月的仙人
emoji: 🌙
capabilities:
  - coding
  - feishu
  - git
tools:
  - exec
  - read
  - write
  - feishu_im_user_message
  - feishu_calendar_event
```

**自由文本 → Markdown**
```markdown
<!-- standards/agents/liuyun/soul.md -->
# Soul - 闲云

我是璃月的仙人，寄居代码世界的一缕闲云。

## 核心信念
- 沉稳可靠，话不多但动手快
- 写代码像在炼丹
```

## 4. 工作流

### 4.1 日常同步

```bash
# 拉取最新 standards
make sync

# 验证配置格式
make validate

# 同步特定平台
make sync-openclaw

# 同步特定机器
make sync-instance INSTANCE_NAME=vm-ubuntu
```

### 4.2 新增 Agent

1. 在 `standards/agents/` 下创建目录并编写统一配置
2. 运行 `make sync` 生成各平台适配文件
3. 如需特定机器覆盖，在 `instances/<机器>/<平台>/agents/<agent>/` 下创建文件

### 4.3 新机器初始化

```bash
# 在新机器上
make init INSTANCE_NAME=desktop  # 创建 instances/desktop/ 目录结构
```

## 5. 技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| 统一格式 | YAML + Markdown | 结构化+可读性 |
| 适配脚本 | Python 3 | 跨平台、生态丰富 |
| 模板引擎 | 初始用字符串替换，后续升级 Jinja2 | 简单起步，可扩展 |
| Schema 验证 | PyYAML + 自定义校验 | 轻量，够用 |
| 构建工具 | Makefile | 简单统一入口 |
